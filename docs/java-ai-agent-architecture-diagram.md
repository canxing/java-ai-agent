### 2.3 数据流图
```
用户输入
    │
    ▼
命令行界面 (CommandLineUI)
    │ 解析输入，判断命令/消息
    ▼
命令处理器 (CommandHandler)
    │ 执行命令或转发消息
    ▼
聊天会话管理器 (ChatSessionManager)
    │ 管理会话状态和上下文
    ▼
配置管理器 (ConfigManager)
    │ 获取API配置和模型参数
    ▼
API客户端工厂 (LLMClientFactory)
    │ 创建合适的客户端实例
    ▼
具体API客户端 (OpenAIClient/ClaudeClient)
    │ 构建HTTP请求，添加认证
    ▼
HTTP客户端 (HttpClient)
    │ 发送请求，处理响应
    ▼
大模型API (OpenAI/Claude/本地)
    │ 处理请求，生成响应
    ▼
响应解析器 (ResponseParser)
    │ 解析JSON响应，提取内容
    ▼
错误处理器 (ErrorHandler)
    │ 检查错误，决定重试/失败
    ▼
对话历史管理器 (ConversationManager)
    │ 保存消息到历史记录
    ▼
响应格式化器 (ResponseFormatter)
    │ 格式化输出，添加颜色等
    ▼
命令行界面 (CommandLineUI)
    │ 显示响应给用户
    ▼
用户看到输出
```

## 3. 关键设计决策

### 3.1 架构风格选择
- **分层架构**：清晰的责任分离，易于维护和测试
- **模块化设计**：高内聚低耦合，便于扩展和重用
- **依赖倒置**：依赖于抽象，不依赖于具体实现

### 3.2 技术选型理由
| 技术 | 选择理由 | 替代方案 |
|------|----------|----------|
| **Java 11+** | 长期支持版本，现代特性，企业级生态 | Kotlin, Scala |
| **Maven** | 标准构建工具，依赖管理成熟 | Gradle |
| **Jackson** | 性能优秀，API友好，社区活跃 | Gson, Moshi |
| **SLF4J** | 日志门面，灵活切换实现 | Log4j2 |
| **JUnit 5** | 现代测试框架，功能丰富 | TestNG |
| **Micrometer** | 指标收集标准，云原生友好 | Dropwizard Metrics |

### 3.3 设计模式应用
1. **工厂模式**：创建不同的大模型客户端
2. **策略模式**：不同的错误处理策略
3. **观察者模式**：会话状态变化通知
4. **装饰器模式**：缓存和日志装饰
5. **建造者模式**：复杂对象构建
6. **命令模式**：命令行命令处理

### 3.4 性能优化策略
1. **连接池**：复用HTTP连接，减少握手开销
2. **缓存**：响应缓存，减少重复请求
3. **异步处理**：非阻塞IO，提高并发能力
4. **批处理**：合并小请求，减少网络往返
5. **压缩**：请求/响应压缩，减少数据传输

### 3.5 容错设计
1. **重试机制**：网络波动时的自动重试
2. **熔断器**：防止级联故障
3. **降级策略**：主服务不可用时的备用方案
4. **超时控制**：防止长时间等待
5. **限流保护**：防止过载
